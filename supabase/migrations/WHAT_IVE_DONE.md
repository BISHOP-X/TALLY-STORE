# Handoff: Crypto Exchange Debugging & Schema Updates

## Context
We were debugging a "Sell Crypto" feature that was failing with various errors (401 Unauthorized, then 400 Bad Request).

## Issues Resolved

### 1. 401 Unauthorized (Edge Function)
- **Problem**: The Edge Function `create-crypto-sell-order` was failing to authenticate the user.
- **Root Cause**: Supabase Edge Functions are stateless. The default `createClient` with `persistSession: true` (default) was trying to use local storage which doesn't exist in the Edge runtime, leading to `AuthSessionMissingError`.
- **Fix**: 
    - Updated `create-crypto-sell-order/index.ts` to initialize `createClient` with `auth: { persistSession: false }`.
    - Updated the code to explicitly extract the `Bearer` token from the `Authorization` header and pass it to `supabase.auth.getUser(token)`.
    - Added robust error handling and debug logging for auth failures.

### 2. 400 Bad Request (Schema Mismatches)
- **Problem**: After fixing auth, the function failed with "Failed to save transaction: Could not find the '...' column".
- **Root Cause**: The `crypto_transactions` table in the database was missing several columns that the Edge Function attempts to write to.
- **Fix**: Created and applied multiple migrations to add the missing columns.

## Applied Migrations
The following migrations were created and pushed to the remote database:

1.  **`20251122120000_add_payment_reference.sql`**
    - Adds `payment_reference` (TEXT)

2.  **`20251122120500_add_more_columns.sql`**
    - Adds NowPayments specific columns:
        - `nowpayments_smart_contract`
        - `burning_percent`
        - `fixed_rate_valid_until`
        - `nowpayments_network`
        - `nowpayments_payin_extra_id`
        - `nowpayments_purchase_id`
        - `nowpayments_pay_address`
        - `nowpayments_amount_received`
        - `outcome_amount`
        - `outcome_currency`
        - `payment_type`
        - `expiration_date`

3.  **`20251122121000_add_rate_column.sql`**
    - Adds `rate` (NUMERIC)
    - Adds `actually_paid` (NUMERIC)

4.  **`20251122121500_add_transaction_type.sql`**
    - Adds `transaction_type` (TEXT)
    - Adds `payment_provider` (TEXT)

## Current State
- **Backend**: The Edge Function `create-crypto-sell-order` is deployed and correctly configured to handle authentication and NowPayments integration.
- **Database**: The `crypto_transactions` table should now have all the necessary columns to store the transaction data generated by the Edge Function.
- **Frontend**: `CryptoExchange.tsx` is currently using a direct `fetch` call (instead of `supabase.functions.invoke`) to provide better visibility into 400/500 error bodies during debugging.
